CSS: CLASSIFY OR IDENTIFY

POSTED SUN 4/19/2015

It turns out that classes and ID's are one of the concepts in CSS that came most naturally to me--so it hopefully it will be easy for me to explain them. Both features allow you to pick out particular elements from your HTML and style them without impacting the rest of the content with that tag so they are dead useful.

THE BASICS

CSS rules are start reference an HTML tag from the document and specifies certain formatting. Here is a CSS rule and a translation:

h1 { font-size= 72px; }
// Take anything from the HTML document tagged with h1 and change the type size to 72 pixels.

What if I have a bunch of h1 titles for different sections that I would like to be different colors or I have one pesky list item that needs to be italicised when the others do not? Well thats where classes and ID's come in. Let's start with classes...

CLASSES

Classes can assigned in the HTML to multiple elements that are receiving the same treatment. Lets say we have a bunch of things on our page that need to purple and centered so we are going to put them in a class called "special". Our HTML might look like this:

<h1 class="special">This text is special</h1>
<p class="special">This text is also special even though its a paragraph and not an h1!</p>
<h1 class="special">I can use this special class again! In fact I can use it as many times as I want in my page.</h1>

.special { color= purple; text-align: center; }

ID's

ID's are unique--as in there is only one on the page. ID's are best for styling exceptions to the rule. Let's say I have a navigation list where each link goes to a page about a specific soccer player. Then right in the middle of it I have to reference a book title that will link to the publisher's site. To indicate to the user that its a book title and not another seciton of my site I want to italicize it.

<li>MESSI</li>
<li>RONALDO</li>
<li id="book">Twenty-Two Foreigners in Funny Shorts</li>
<li>ZLATAN</li>

#book { font-style= italic; }

THEY'RE NOT JUST FOR CSS

This is just the beginning of what classes and ID's are handy for. Once we get into JavaScript they are used for all sorts of things becuas they can target particular elements on the page with a high level of specificity.
